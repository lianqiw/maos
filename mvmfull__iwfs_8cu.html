Detailed Description

Test MVM for a single WFS. Using two GPUs. Optional using networking. ethtest is the server This is not part of maos executable. Called by test/test_gpu executable.

two ways: 1) broadcast gradients to both GPUs with each GPU handle part of actuators. (not tested yet) 2) partition gradients to GPUs with GPU handle all actuators. (selected)

Use schedtool -a 0x1 PID to let the exe only run one CPU 0. This prevents jitter.

Optimal timing (low jitter) achieved with multimv_do: cassiopeia with GTX 580:nsm=1, mtch_ngrid=90, nover=9 1.1ms orion with single GTX 590: nsm=1, mtch_ngrid=90, nover=9, 1.3ms kepler with K20: nsm=2, mtch_ngrid=20, nover=3 1.13ms cassiopeia with single GTX590

A standalone routine that testes applying MVM for a single WFS and update mvm. The orderig of gradients are like xyxyxy instead of normal xxxyyy.

Important: 1) Only page locked host memory can do async memcpy that overallps with computation 2) Has to be Portable for multiple GPUs to do async memcpy concurrently.
